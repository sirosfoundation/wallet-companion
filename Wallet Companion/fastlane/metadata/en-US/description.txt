Wallet Companion is a support layer between your browser and web-based digital identity wallets.
Its job is to make wallets work better across the web as the ecosystem matures, while keeping your data secure, private, and yours.
The extension is still in beta, and is being actively developed.

# How it works:

You register the web wallets* you use, either through the Wallet Companion options page or from the wallet itself.
Now, Wallet Companion's features are available to support that wallet.
Today, it enables your wallet to receive requests over the Digital Credentials API.


# Digital Credentials API support, everywhere

The first major feature of Wallet Companion is a web-native implementation of OpenID For Verifiable Presentations over the W3C Digital Credentials API.
When a site asks you to prove something, (your identity, your age, a membership) you choose which of your registered wallets responds, and Wallet Companion sends the result back to the site.

Any site using the Digital Credentials API works without special integration. Wallets need to support Wallet Companion. The SIROS ID Wallet does today, and other wallet developers can add support via our open API.

More wallet-interoperability features will be added in future releases.


# Key points:

- Compatible web wallets and verifiers benefit from DC API discovery, on the web.
- Works on major browsers and normal sites.
- Increased privacy when sharing credentials: Wallet Companion talks directly with issuers and verifiers, so your data stays on your device until you're ready to share it.


# Getting started:

1. Install Wallet Companion
2. Visit a supported web wallet, currently SIROS ID*, it registers automatically
3. When a site requests credentials, pick your wallet from the popup
4. Approve the request in your wallet
